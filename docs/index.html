    Ingenico One Web Service API (WSAPI) Documentation

    API Endpoint

    https://domain.com

    The Ingenico One WSAPI platform is a set of REST APIs designed to be consumed by developers for interaction with our Mobile Commerce Manager (MCM). This provides access to all of the features offered for accepting payments, performing administrative tasks, and conducting full end-end transaction processing.

    API Library

    Please reference this section for the detailed view of the APIs provided within the Ingenico One WSAPI platform.

    Authentication

    Authenticate your account by including your application token in the request field X-Roam-Key. If the credentials and token are valid, the API will return a response with the necessary session token information. This session token can be used for accessing other APIs available within our library.

    You will receive a session token by logging in

    Ingenico uses application tokens to allow access to the API suite. You can register a new application token at our developer portal.

    Please replace X-Roam-Key in the header with your application token.

    Log in

    URL

    POST [domain]/wsapi/Authentication/

    First, you must log in to your account using your application token and assigned credentials. As stated earlier, this will return your session token along with other various user settings.

    Headers

    Header

    
X-Roam-Key: "APP6-54a59a59ea7-7440-4f8b-ae96-ffba2e801391"
X-Roam-ApiVersion: "6.10.0"
X-Roam-ClientVersion: "1.0.0"
Content-Type: "application/json"
    Parameter Required Description
    X-Roam-Key Yes Application token that identifies the request origin. This key is unique to every client, as well as every environment. Please contact API Support (APIsupport.us@ingenico.com) to get this key.
    X-Roam-ApiVersion Yes Version of the API being used (e.g. 6.10.0)
    X-Roam-ClientVersion Yes Client version which can be vary by application. This is set by developers (e.g. 1.0.0)
    Content-Type Yes Content type e.g. application/json

    Request

    Pass the following values in the request body to successfully log in.

    Request

    {
  user_name:"myUserName",
  password:"pwd123"
}
    Parameter Type Required Description
    user_name string Yes User's unique identifier
    password string Yes User's password

    Refresh a Session

    URL

    PUT [domain]/wsapi/Authentication/

    This call is used to keep a logged-in session alive. This API method will refresh a valid session and extend the session token expiration time.

    Headers

    Header

    
X-Roam-Token: "MCM6-420ba742-ee06-401a-b1a6-0f6054a3e20d"
X-Roam-ApiVersion: "6.10.0"
X-Roam-ClientVersion: "1.0.0"
Content-Type: "application/json"
    Parameter Required Description
    X-Roam-Token Yes Session token from the log in response. We use this to identify the API caller and its permissions.
    X-Roam-ApiVersion Yes Version of the API being used (e.g. 6.10.0)
    X-Roam-ClientVersion Yes Client version which can be vary by application. This is set by developers (e.g. 1.0.0)
    Content-Type Yes Content type e.g. application/json

    Request

    A request body is not needed to refresh a session.

    Request
    Parameter Type Required Description
    N/A

    Response

    If your request is successful, you will receive a response with the following outputs defined:

    Response:

    {
  chain_id: "679",
  store_id: "3",
  terminal_id: "1",
  organization_type: "merchant",
  session: {
    expires: "20151222175141",
    session_token: "MCM6-420ba742-ee06-401a-b1a6-0f6054a3e20d"
  },
  user: {
    user_name: "myUserName",
    is_admin: false,
    is_sys_admin: false,
    test_account: false
  }
}
    Parameter Type Required Description
    chainID string Yes Gateway identifier for specifying processor together with store_id.
    store_id string Yes Gateway identifier for specifying processor together with chain_id
    terminal_id string Yes MCM identifier for device.
    organization_type object Yes The type of the user's parent organization.
    session object Yes Session info including token and expiration information.
    user object Yes Information for the authenticated user.

    Log out

    URL

    DELETE [domain]/wsapi/Authentication/

    This call is used to log the user out of an active session, invalidating the existing session token.

    Headers

    Header

    
X-Roam-Token: "MCM6-420ba742-ee06-401a-b1a6-0f6054a3e20d"
X-Roam-ApiVersion: "6.10.0"
X-Roam-ClientVersion: "1.0.0"
Content-Type: "application/json"
    Parameter Required Description
    X-Roam-Token Yes Session token from the log in response. We use this to identify the API caller and its permissions.
    X-Roam-ApiVersion Yes Version of the API being used (e.g. 6.10.0)
    X-Roam-ClientVersion Yes Client version which can be vary by application. This is set by developers (e.g. 1.0.0)
    Content-Type Yes Content type e.g. application/json

    Request

    A request body is not needed to log out of a session.

    Request
    Parameter Type Required Description
    N/A

    Response

    Response:

    HTTP Status Code 200 OK
    HTTP Status code: 200 OK denotes success
